# ชีวะกับพี่อาร์ — Tutor Pocket (Supabase Edition)

ระบบจัดการติวเตอร์ฉบับปรับปรุงใหม่ ทำงานเป็นไฟล์ `index.html` ไฟล์เดียว ใช้ **Supabase** เป็นฐานข้อมูล
(แทน Google Apps Script เดิม) และมี **ระบบล็อกอิน (Supabase Auth)** ป้องกันข้อมูลก่อนที่จะ deploy ขึ้น GitHub Pages

ไฟล์ที่ได้รับ:
- `index.html` — ตัวแอปทั้งหมด (อัปโหลดขึ้น GitHub Pages ได้เลย)
- `supabase_schema.sql` — คำสั่งสร้างตาราง/ระบบความปลอดภัยใน Supabase
- `README.md` — ไฟล์นี้

---

## ขั้นตอนที่ 1: สร้างโปรเจกต์ Supabase

1. ไปที่ [supabase.com](https://supabase.com) → สมัคร/เข้าสู่ระบบ → **New Project**
2. ตั้งชื่อโปรเจกต์ ตั้งรหัสผ่านฐานข้อมูล (เก็บไว้ให้ดี) แล้วเลือก Region ที่ใกล้ที่สุด (เช่น Singapore)
3. รอสักครู่ให้โปรเจกต์สร้างเสร็จ

## ขั้นตอนที่ 2: รันคำสั่งสร้างฐานข้อมูล

1. ในเมนูซ้ายของ Supabase Dashboard ไปที่ **SQL Editor** → **New query**
2. เปิดไฟล์ `supabase_schema.sql` ที่แนบมา คัดลอกทั้งหมด แล้ววางในช่อง SQL Editor
3. กด **Run** (รันครั้งเดียวพอ คำสั่งเขียนแบบรันซ้ำได้โดยไม่พัง หากจำเป็นต้องรันใหม่ในอนาคต)
4. จะได้ตารางทั้งหมด 9 ตาราง พร้อมระบบความปลอดภัยระดับแถว (RLS) และ Realtime sync เปิดใช้งานอัตโนมัติ

## ขั้นตอนที่ 3: สร้างบัญชีผู้ใช้สำหรับล็อกอิน

เนื่องจากแอปนี้ออกแบบมาสำหรับติวเตอร์ **คนเดียว** จึงไม่มีหน้าสมัครสมาชิกสาธารณะ ต้องสร้างบัญชีเองผ่าน Dashboard:

1. ไปที่ **Authentication → Users → Add user → Create new user**
2. กรอกอีเมลและรหัสผ่านที่จะใช้ล็อกอิน (เลือก "Auto Confirm User" ให้ติ๊กถูกไว้ด้วย จะได้ไม่ต้องยืนยันอีเมล)
3. กด **Create user**
4. (แนะนำ) ไปที่ **Authentication → Providers → Email** แล้วปิด **"Allow new users to sign up"**
   เพื่อป้องกันไม่ให้คนอื่นสมัครสมาชิกเข้าระบบเองได้ — เพราะแอปนี้ตั้งใจให้มีผู้ใช้แค่คนเดียว

## ขั้นตอนที่ 4: นำ Project URL และ Anon Key มาใส่ในไฟล์ index.html

1. ไปที่ **Project Settings → API**
2. คัดลอกค่า **Project URL** และ **anon public key**
3. เปิดไฟล์ `index.html` ด้วยโปรแกรมแก้ไขข้อความ (เช่น Notepad, VS Code) ค้นหาส่วนนี้ใกล้ๆ ด้านบนของไฟล์:

```javascript
const SUPABASE_URL = "https://YOUR-PROJECT-REF.supabase.co";
const SUPABASE_ANON_KEY = "YOUR-ANON-PUBLIC-KEY";
```

4. แทนที่ค่าทั้งสองด้วยค่าจริงของคุณ แล้วบันทึกไฟล์

> **หมายเหตุ:** การใส่ `anon key` ตรงๆ ในไฟล์ HTML เป็นวิธีมาตรฐานของ Supabase สำหรับเว็บฝั่ง client
> คีย์นี้ถูกออกแบบมาให้เปิดเผยได้ ความปลอดภัยจริงๆ มาจากระบบ Row Level Security (RLS) ที่ตั้งไว้ในขั้นตอนที่ 2
> ซึ่งจะอนุญาตให้เข้าถึงข้อมูลได้เฉพาะบัญชีที่ล็อกอินและเป็นเจ้าของข้อมูลแถวนั้นๆ เท่านั้น

## ขั้นตอนที่ 5: Deploy ขึ้น GitHub Pages

1. สร้าง repository ใหม่บน GitHub (หรือใช้ของเดิม)
2. อัปโหลดไฟล์ `index.html` ขึ้นไปที่ root ของ repository (หรือโฟลเดอร์ `/docs` ก็ได้)
3. ไปที่ repository → **Settings → Pages**
4. เลือก Source เป็น branch และโฟลเดอร์ที่ไฟล์ `index.html` อยู่ แล้วกด **Save**
5. รอสักครู่ GitHub จะให้ลิงก์เว็บไซต์ เช่น `https://your-username.github.io/repo-name/`

## ขั้นตอนที่ 6: ทดสอบใช้งาน

1. เปิดลิงก์เว็บไซต์ที่ได้
2. หน้าจอล็อกอินจะขึ้นมา — กรอกอีเมล/รหัสผ่านที่สร้างไว้ในขั้นตอนที่ 3
3. เข้าสู่ระบบสำเร็จ → เริ่มใช้งานได้ทันที
4. แนะนำให้ไปที่แท็บ **ตั้งค่า** เพื่อกรอกข้อมูลสถาบัน/บัญชีธนาคาร/พร้อมเพย์ ให้ครบก่อนเริ่มออกใบเสร็จ/ใบแจ้งหนี้จริง

## สิ่งที่ปรับปรุงจากเวอร์ชันเดิม

**เปลี่ยนระบบหลังบ้าน**
- ย้ายจาก Google Apps Script มาเป็น Supabase (Postgres) พร้อม Row Level Security — ข้อมูลของแต่ละบัญชีแยกจากกันโดยสมบูรณ์
- เพิ่มระบบล็อกอิน (Supabase Auth) ป้องกันข้อมูลนักเรียน/การเงินก่อนเปิดเป็นเว็บสาธารณะ
- เพิ่ม Realtime sync — เปิดใช้งานพร้อมกันได้หลายอุปกรณ์ ข้อมูลอัปเดตให้กันอัตโนมัติ

**แก้บั๊กที่พบ**
- แท็บ "คอร์สเรียน" เดิมมีโค้ดซ้ำซ้อนทำให้เนื้อหาแสดงสองรอบ — แก้แล้ว
- การจัดเรียงรายการ "สิ่งที่ต้องทำ" เดิมแก้ไข state array ตรงๆ ระหว่าง render (อาจทำให้ลำดับเพี้ยนได้) — แก้เป็นการสร้างอาร์เรย์ใหม่ก่อน sort
- ช่อง "พร้อมเพย์" เดิมกรอกแล้วหายเมื่อรีเฟรชหน้า (ไม่เคยถูกบันทึกจริง) — ตอนนี้บันทึกลงฐานข้อมูลถาวรแล้ว
- ข้อมูลชื่อ/ที่อยู่/เลขบัญชีในใบเสร็จ-ใบแจ้งหนี้ เดิม hardcode ไว้ในโค้ด — ย้ายมาเป็นฟิลด์แก้ไขได้ในหน้าตั้งค่า

**ฟีเจอร์ที่เพิ่มเข้ามา**
- ลบ/แก้ไขรายการได้ครบทุกส่วน (เดิมไม่มีปุ่มลบให้ รายการการเงิน / บันทึกการสอน / คะแนนสอบ / เอกสาร เลย)
  - ลบบันทึกการสอน จะคืนชั่วโมงเรียนให้นักเรียนอัตโนมัติ เพื่อให้ข้อมูลชั่วโมงถูกต้องเสมอ
- กล่องยืนยันก่อนลบ (แทน popup เดิมของเบราว์เซอร์) ให้ดูเป็นมืออาชีพขึ้น
- สำรอง/กู้คืนข้อมูลทั้งระบบเป็นไฟล์ JSON ได้จากหน้าตั้งค่า (เผื่อกรณีฉุกเฉิน)
- แสดงสถานะออนไลน์/ออฟไลน์ และอีเมลผู้ใช้ที่ล็อกอินอยู่ที่หัวเว็บ พร้อมปุ่มออกจากระบบ
- หน้าจอแจ้งเตือนกันพังหากยังไม่ได้ตั้งค่า Supabase หรือเกิดข้อผิดพลาดไม่คาดคิด (แทนหน้าจอขาวๆ)

**ตั้งใจไม่ทำในรอบนี้ (เพื่อคุมขอบเขตและความเสี่ยง)**
- โหมดกลางคืน (Dark mode) — ต้องปรับ class จำนวนมากทั่วทั้งแอป หากต้องการแจ้งได้ในรอบถัดไป
- แถบเมนูล่างสำหรับมือถือ — เมนูบนแบบเลื่อนแนวนอนเดิมรองรับมือถือได้ดีอยู่แล้ว
- การแก้ไขบันทึกการสอน/คะแนนสอบ (รองรับแค่ลบ+เพิ่มใหม่) — เพราะการแก้ไขต้องคำนวณชั่วโมงเรียนใหม่ซับซ้อนกว่า

## แก้ปัญหาเบื้องต้น

- **หน้าจอค้างที่ "ยังไม่ได้ตั้งค่า Supabase"** → ตรวจสอบว่าใส่ `SUPABASE_URL`/`SUPABASE_ANON_KEY` ถูกต้องและไม่มีช่องว่างเกิน
- **ล็อกอินไม่ได้ / อีเมลหรือรหัสผ่านไม่ถูกต้อง** → ตรวจสอบว่าสร้างผู้ใช้ใน Authentication → Users แล้ว และติ๊ก Auto Confirm ไว้
- **บันทึกข้อมูลไม่ได้ / ขึ้น error** → ตรวจสอบว่ารันไฟล์ `supabase_schema.sql` ครบแล้ว และเชื่อมต่ออินเทอร์เน็ตอยู่ (มีป้ายสถานะออนไลน์/ออฟไลน์ที่หัวเว็บ)
- **ข้อมูลไม่ sync ข้ามอุปกรณ์** → ตรวจสอบว่าใน Supabase มีตารางที่เกี่ยวข้องอยู่ใน Realtime publication แล้ว (สคริปต์ SQL เปิดให้อัตโนมัติ)
